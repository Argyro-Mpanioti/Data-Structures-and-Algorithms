•main.cpp

In main.cpp the file commands.txt is read and the commands written in it are executed. Specifically, each line of commands.txt is read and saved in the string "line" until the command that the line contains is executed. When the command gets executed, the process is repeated for the rest of the lines of until there are no other commands in commands.txt.
The content of line is saved in the character array "com". This is done to allow the use of the function strstr later on. After this, there is an if...else if structure that checks which command is in com. According to the command the appropriate actions are executed. The execution time is counted and written in the "output.txt" file.

•node

In order to explain how the class avltree works we should first explain how the class node works. Each object of "node" is a node of an avltree.

The class node has 4 attributes:
> value: integer that contains the value of the node
> lc: a pointer to the node that contains the left child. If the node doesn't have a left child then the pointer points to NULL.
> rc: a pointer to the node that contains the right child. If the node doesn't have a right child then the pointer points to NULL.
> pr: a pointer to the node that contains the parent. If the node doesn't have a parent (so if the node is the root) then the pointer points to NULL.

The class node has the methods:
> A default constructor that makes the value of the node 0 and makes the children pointers point to NULL
> A constructor that gets as a parameter an integer and makes it the value of the node and makes the pointers to the children point to NULL
> Setters and getters for the private attributes

In the "insert" method of avltree:

The insertion of an element in the AVL tree is split in 2 parts:
1) Insertion of the element in the tree in a way that allows the tree to remain a binary search tree.
2) Execution of the necessary rotations so that the tree remains an AVL tree.

1) Insertion of the element in the tree in a way that allows the tree to remain a binary search tree:
If the element that we wish to insert is the first element of the tree then we make it the root.
Otherwise, we search for the parent of the element starting from the root.

2) Execution of the necessary rotations so that the tree remains an AVL tree:
For the execution of these rotations the method rotations is called. The parameter of the function is the pointer of the node we inserted.

The method rotations gets as a parameter a pointer to a node of the tree. The method has a the boolean variable "balanced". balanced is true when we do the rotation. Η συνάρτηση ελέγχει αν χρειάζεται κάποια περιστροφή το δένδρο που ξεκινάει από τον κόμβο που δέχεται ή από κάποιον πρόγονό του και εκτελεί αυτήν την περιστροφή.

Συγκεκριμένα, αν ο κόμβος είναι κενός ή έχει εκτελεστεί η απαραίτητη περιστροφή για να γίνει το δένδρο avl τότε επιστρέφει. Αλλιώς, υπολογίζει το ύψος των δύο υποδένδρων του με κλήση της συνάρτησης foo (η λειτουργία της foo εξηγείται στη συνέχεια).

Αν το ύψος του δεξιού υποδένδρου του κόμβου είναι μεγαλύτερο από το ύψος του αριστερού υποδένδρου του κόμβου και η διαφορά τους είναι μεγαλύτερη ή ίση με 2 τότε χρειάζεται να γίνει απλή αριστερή περιστροφή ή διπλή περιστροφή RL. Αν το ύψος του δεξιού υποδένδρου του δεξιού παιδιού του κόμβου είναι μεγαλύτερο από το ύψος του αριστερού υποδένδρου του δεξιού παιδιού του κόμβου τότε γίνεται απλή αριστερή περιστροφή. Αλλιώς γίνεται διπλή περιστροφή RL. Αν γίνει κάποια από τις δύο περιστροφές εκχωρούμε στην balanced την τιμή true.

Αν το ύψος του αριστερού υποδένδρου του αριστερού παιδιού του κόμβου είναι μεγαλύτερο από το ύψος του δεξιού υποδένδρου του παιδού του κόμβου τότε γίνεται απλή δεξιά περιστροφή. Στην αρχή έχουμε 2 μεταβλητές bool, την root και την right. Κάνουμε την root true αν ο κόμβος που βρήκαμε με την αναζήτηση είναι ρίζα. Κάνουμε το deji true αν ο κόμβος που βρήκαμε με την αναζήτηση είναι δεξί παιδί του γονιού του.

If the node is a leaf:
1st step:
If the node is the root then we make the root point to NULL.
Αλλιώς, αν ο κόμβος είναι δεξί παιδί του γονέα του, τότε το δεξί παιδί του γονέα γίνεται το NULL.
Αλλιώς, αν ο κόμβος είναι αριστερό παιδί του γονέα του, τότε το αριστερό παιδί του γονέα γίνεται το NULL.
2nd step:
Καλείται η συνάρτηση rotations με παράμετρο τον δείκτη στον γονέα του κόμβου που διαγράφεται. Αυτό δεν χρειάζεται να γίνει σε περίπτωση που ο κόμβος είναι και φύλλο και ρίζα γιατί τότε δεν υπάρχουν άλλα στοιχεία.

If the node only has a right subtree:
1st step:
If the node is the root then the right child of the node becomes the root.
Otherwise, if the node is the right child of its parent, then the right child of the node becomes the right child of the node's parent.
Otherwise, the node is the left child of its parent. Then, the right child of the node becomes the left child of the node's parent.
2nd step:
The method "rotations" is called. The parameter it takes is the pointer to the right child of the node we are deleting.

Αλλιώς, αν ο κόμβος έχει μόνο αριστερό υποδένδρο:
1ο βήμα:
Αν ο κόμβος είναι ρίζα τότε γίνεται ρίζα το αριστερό παιδί του κόμβου.
Αλλιώς, αν ο κόμβος είναι δεξί παιδί του γονέα του, τότε το αριστερό παιδί του κόμβου γίνεται δεξί παιδί του γονέα του κόμβου.
Αλλιώς, αν ο κόμβος είναι αριστερό παιδί του γονέα του, τότε το αριστερό παιδί του κόμβου γίνεται αριστερό παιδί του γονέα του κόμβου.
2ο βήμα:
Καλείται η συνάρτηση rotations με παράμετρο τον δείκτη στο αριστερό παιδί του κόμβου που διαγράφεται.

Αλλιώς, αν ο κόμβος έχει αριστερό και δεξί υποδένδρο:
1st step:
The node is replaced by the smallest element of its right subtree and the left child of the element's parent points to NULL. But if the right subtree only contains 1 element then the parent of the smallest element of the right subtree is the node itself. So, in this case the right child points to NULL, not the left one.
2nd step:
Καλείται η συνάρτηση rotations με παράμετρο τον δείκτη στο νέο μικρότερο στοιχείο του δεξιού υποδένδρου του κόμβου που διαγράφεται.

Also, the size of the AVL tree is reduced by 1 when the deletion is done.


•vertex

In order to explain how the graph works, we should first explain how the class "vertice" works. Each object of "vertex" is a vertex of the graph.

The class vertex has 6 attributes:
> value: an integer that contains the value of the vertex.
> edges: a dynamic array that contains all the vertices with whom the vertex is connected.
> edges_count: an integer that contains the amount of προσπίπτουσων edges of the vertex.
> weights: a dynamic array of integers that contains the weights of all the edges με τις οποίες είναι συνδεδεμένη η συγκεριμένη κορυφή. The arrays edges and weights are parallel arrays.
> Τον δυναμικό πίνακα ακεραίων edges_positions στον οποίο αποθηκεύονται οι θέσεις όλων των κορυφών με τις οποίες συνδέεται η συγκεριμένη κορυφή στον πίνακα vertices. Ο πίνακας th_edges είναι παράλληλος με τους πίνακες edges και weights.
> Την ακέραια μεταβλητή position_in_vertices στην οποία αποθηκεύεται η θέση της κορυφής στον πίνακα vertices.

The methods of class "vertice" are:
> A default constructor, that initializes the value of the vertex as 0 and the count of προσπίπτουσων edges of the vertex as 0.
> Έναν κατασκευαστή με παράμετρο μία ακέραια μεταβλητή. Ο κατασκευαστής την κάνει τιμή της κορυφής και αρχικοποίει το πλήθος των προσπίπτουσων ακμών της κορυφής με 0.
> Έναν copy constructor.
> Τη συνάρτηση insert που συνδέει την κορυφή για την οποία εκτελείται με την κορυφή που δέχεται ως παράμετρο. Η ακμή που τις συνδέει έχει ως βάρος την τιμή που δέχεται η συνάρτηση ως 2η παράμετρο. Υποθέτουμε ότι οι δύο κορυφές δεν είναι ήδη συνδεδεμένες.
> Τη συνάρτηση delete_edge που δέχεται ως παράμετρο μία κορυφή και διαγράφει από τον γράφο την ακμή που τη συνδέει με την κορυφή για την οποία εκτελείται η συνάρτηση.
> setters και getters.
> Υπερφόρτωση του τελεστή = έτσι ώστε αυτός να λειτουργεί και για κορυφές.

Αναλυτικότερα στην insert ενημερώνουμε τους πίνακες edges,weights,th_edges αυξάνοντας το μέγεθός τους κατά 1 και προσθέτοντας την νέα γειτονική κορυφή, το βάρος της νέας ακμής και την θέση της στον πίνακα edges σε αυτούς αντίστοιχα. Επίσης, αυξάνουμε την ακέραια μεταβλητή pl_edges.

Η delete_edge αρχικά ελέγχει αν η ακμή που συνδέει την κορυφή με την κορυφή που δέχεται ως 1η παράμετρο υπάρχει στον γράφο και αν βρει ότι η ακμή υπάρχει αποθηκεύει την θέση που έχει στους πίνακες edges/weights/th_edges (είναι η ίδια γιατί οι πίνακες είναι παράλληλοι) στην μεταβλητή position. Αν η ακμή δεν υπάρχει εκτελείται η return, αλλιώς ενημερώνουμε τους πίνακες edges,weights,th_edges αφαιρώντας από αυτούς το στοιχείο στην θέση thesh και μειώνοντας το μέγεθός τους κατά 1.

In main, when a line of αρχείου commands.txt εντοπιστεί η φράση INSERT GRAPH τότε οι 3 συμβολοσειρές που ακολουθούν  μετατρέπονται σε ακέραιους και καλείται η insert για το αντικείμενο gr που είναι ο γράφος μας με παραμέτρους τις 3 αυτές τιμές n1,n2 και w.

Στην delete_graph_edge αναζητούμε τις 2 κορυφές για να βρούμε αν υπάρχουν στον γράφο. Αν και οι δύο υπάρχουν τότε καλούμε την delete_edge της vertex για την 1η κορυφή με παράμετρο την 2η κορυφή και για την 2η κορυφή με παράμετρο την 1η κορυφή. Επίσης, μειώνουμε το μέγεθος του γράφου κατά ένα.



-COMPUTESHORTESTPATH GRAPH number1 number2

Στη main όταν σε κάποια γραμμή του αρχείου commands.txt εντοπιστεί η φράση COMPUTESHORTESTPATH GRAPH number1 number2 τότε μετατρέπουμε τις δύο συμβολοσειρές που ακολουθούν την φράση COMPUTESHORTESTPATH GRAPH σε ακέραιες και τις αποθηκεύουμε στις ακέραιες μεταβλητές n1 και n2. Έπειτα καλείται η συνάρτηση Dijkstra για τον γράφο με παραμέτρους τις τιμές n1,n2 και το αποτέλεσμά της εγγράφεται στο αρχείο output.txt.

Στη συνάρτηση Dijkstra χρησιμοποιείται μία επιπλέον κλάση η οποία μοιάζει με την κλάση του σωρού ελαχίστων. Η κλάση αυτή ονομάζεται minheap_gr. Η κλάση αυτή λειτουργεί σαν τον σωρό ελαχίστων που έχουμε ήδη εξηγήσει. Οι διαφορές τους αναφέρονται παρακάτω. Αρχικά, η minheap_gr αντί να δέχεται ως τιμές των κόμβων ακέραιους αριθμούς, δέχεται κόμβους του γράφου τους οποίους αποθηκεύει σε έναν δυναμικό πίνακα v. Επίσης, έχει έναν ακέραιο δυναμικό πίνακα keys. Οι τιμές του σωρού διατάσσονται με βάση τις τιμές του πίνακα keys. Επίσης έχει έναν δυναμικό πίνακα th_ston_k στον οποίο αποθηκεύονται οι θέσεις των κορυφών του πίνακα v στον πίνακα koryfes. Οι 3 δυναμικοί πίνακες είναι παράλληλοι.

Η κλάση minheap_gr έχει την συνάρτηση decreasekey, η οποία εντοπίζει την κορυφή που δέχεται ως πρώτη παράμετρο στον σωρό, και σε εκείνο το στοιχείο του σωρού αποθηκεύει στο keys την 2η παράμετρο. Στη συνέχεια η συνάρτηση φροντίζει έτσι ώστε ο σωρός να παραμείνει σωρός ελαχιστων με την κλήση των συναρτήσεων sortdeleted και sort.

Η κλάση minheap_gr έχει και την αναδρομική συνάρτηση sort η οποία δέχεται ως παράμετρο μία ακέραια μεταβλητή th. Η sort φροντίζει έτσι ώστε κάθε φορά ο γονέας του στοιχείου της θέσης th στον σωρό να έχει μικρότερή τιμή keys από την τιμή keys του παιδιού. Αν αυτό δεν ισχύει τότε τα αντιμεταθέτει και επαναλαμβάνει τον έλεγχο για το στοιχείο που είναι πλέον γονέας. Αλλιώς επιστρέφει.

Η συνάρτηση Dijkstra δέχεται δύο ακέραιες μεταβλητές n1,n2 ως παραμέτρους. Η συνάρτηση υπολογίζει και επιστρέφει το κόστος της ελάχιστης διαδρομής από τον κόμβο με τιμή n1 μέχρι τον κόμβο με τιμή n2. Η συνάρτηση Dijkstra έχει τον ακέραιο πίνακα d στον οποίο αποθηκεύει τις αποστάσεις όλων των κόμβων του γράφου από τον κόμβο με τιμή n1.

Στην συνάρτηση κάθε κόμβος του γράφου εισάγεται στον σωρό ελαχίστων με 2η παράμετρο του το στοιχείο d που αντιστοιχεί σε αυτόν και 3η παράμετρο την θέση του στον πίνακα koryfes. Ο Dijkstra λειτουργεί ως εξής: κάθε φορά διαγράφεται η ρίζα από τον σωρό και ενημερώνεται ο πίνακας d αν με το στοιχείο που διαγράφηκε από τον σωρό μειώνεται κάποια από τις αποστάσεις του κόμβου με τιμή n1 προς κάποιον γειτονικό κόμβο του στοιχείου αυτού που διαγράφεται από τον σωρό

-FINDCONNECTEDCOMPONENTS GRAPH

In main, when the phrase FINDCONNECTEDCOMPONENTS GRAPH is detected in a line of "commands.txt" then the method fcc of the graph is called and the value it returns is written in the file "output.txt".

In the method fcc there is the array "color" in which the color of each κορυφή is saved. If we haven't visited the κορυφή yet, the color is 'w'. While we are visiting it th color is 'g'. After we have vistied it the color is 'b'. Initially, the color of each κορυφών is 'w'. Για κάθε κορυφή, αν δεν την έχουμε επισκεπτεί, αυξάνουμε το πλήθος των συνεκτικών συνιστωσών κατά 1 και επισκεπτόμαστε αυτή την κορυφή και κάθε κορυφή της συνεκτικής συνιστώσας στην οποία ανήκει. Αυτό γίνεται με τη συνάρτηση DFSVisit.

Η αναδρομική συνάρτηση DFSVisit δέχεται ως παράμετρο τον πίνακα color και μία ακέραια μεταβλητή u και προσπελάυνει τα στοιχεία της συνιστώσας που περιέχει την κορυφή που βρίσκεται στην θέση u στον πίνακα koryfes. Η συνάρτηση ενημερώνει και τον πίνακα color.



-COMPUTESPANNINGTREE GRAPH

Στη main όταν σε κάποια γραμμή του αρχείου commands.txt εντοπιστεί η φράση COMPUTESPANNINGTREE GRAPH τότε καλείται η συνάρτηση cc2 για τον γράφο και το αποτέλεσμά της, που είναι το κόστος του ελάχιστου εκτεινόμενου δένδρου του γράφου, εγγράφεται στο αρχείο output.txt.

The method cc2 helps in the execution of the Prim algorithm for each συνιστώσα of the graph. Η συνάρτηση έχει έναν πίνακα color2 στον οποίο έχει αποθηκευμένο το χρώμα κάθε κορυφής. Αν δεν έχουμε επισκεπτεί την κορυφή τότε το χρώμα της είναι 'w'.
Όσο την επισκεπτόμαστε το χρώμα της είναι 'g'. Αφού την επισκεπτούμε το χρώμα της είναι 'b'. Για κάθε κορυφή του γράφου αν δεν την έχουμε επισκεπτεί τότε έχουμε έναν πίνακα color3, τον οποίο αρχικοποιούμε με 'w', και καλούμε τη συνάρτηση DFSVisit2 με παραμέτρους τον color2, την θέση της κορυφής στον πίνακα koryfes και τον color3. Η DFSVisit2 λειτουργεί σαν την DFSVisit μόνο που ενημερώνει και τον πίνακα color3. Άρα, ο color3 μετά την εκτέλεση της DFSVisit2 περιέχει την τιμή 'b' μόνο για τις κορυφές της συνεκτικής συνιστώσας στην οποία ανήκει η κορυφή της θέσης i του πίνακα koryfes.

cc2 returns the cost of the minimum spanning tree that it computes with the variable "result". Η result αρχικοποιείται με 0 και μετά από κάθε εκτέλεση του DFSVisit2 ενημερώνεται. Κάθε φορά η result αυξάνεται κατά την τιμή που επιστρέφει η συνάρτηση Prim με παραμέτρους τον πίνακα color3 και την θέση i.

The method Prim has 2 arrays: "color" and "key". Αρχικά, οι πίνακες αρχικοποιούνται, ο color με 'w' και ο key με τον μεγαλύτερο δυνατό ακέραιο (INT_MAX). Έχουμε έναν σωρό ελαχίστων τύπου minheap_gr στον οποίο εισάγουμε κάθε κόμβο του γράφου με το αντίστοιχο key και τη θέση του στον πίνακα koryfes.

Όσο ο σωρός έχει στοιχεία διαγράφουμε τη ρίζα του και την αποθηκεύουμε στην κορυφή u. Για κάθε γειτονική κορυφή της u αν δεν την έχουμε επισκεπτεί και μπορούμε να μειώσουμε το key του u μέσω αυτής τότε ενημερώνουμε το key και τον σωρό.


Η συνάρτηση έχει μία long int μεταβλητή weight που είναι το βάρος του ελάχιστου εκτεινόμενου δένδρου της συνεκτικής συνιστώσας που περιέχει τα στοιχεία τα οποία έχουν color3 'b'. Το weight θα γίνει το άθροισμα των key τα οποία έχουν color3 'b'. Η συνάρτηση επιστρέφει το weight.



•hashtable


Έστω οτι η συνάρτηση h(n)=(3*n+7) mod maxsize είναι η συνάρτηση κατακερματισμού μας.

-BUILD HASHTABLE filename

Για τον πίνακα κατακερματισμού στην main ακολουθούνται τα ίδια βήματα που ακολουθούνται και στις BUILD των προηγούμενων δομών. 

Η κλάση hashtable έχει 2 μεταβλητές:
> Τον δυναμικό πίνακα ακεραίων data ο οποίος είναι ο πίνακας κατακερματισμού μας
> Την ακέραια μεταβλητή maxsize, η οποία είναι το μέγεθος του πίνακα data

Η κλάση hashtable έχει τον default κατασκευαστή που αρχικοποιεί το maxsize με 50 και κάνει το data πίνακα με 50 θέσεις.

Η κλάση hashtable έχει επίσης μία συνάρτηση build που δέχεται ως παράμετρο το όνομα του αρχείου από το οποίο διαβάζει τα δεδομένα για την κατασκευή του πίνακα κατακερματισμού και τα εισάγει στον πίνακα κατακερματισμού. Αυτό γίνεται με την κλήση της insert για κάθε τιμή που διαβάζεται από το αρχείο. Η τιμή αυτή γίνεται παράμετρος της insert.



-INSERT HASHTABLE number

Στη main όταν σε κάποια γραμμή του αρχείου commands.txt εντοπιστεί η φράση INSERT HASHTABLE τότε η συμβολοσειρά που ακολουθεί μετατρέπεται σε ακέραιο όπως έγινε και στις INSERT των προηγούμενων δομών και καλείται η insert για το αντικείμενο ht με παράμετρο τον ακέραιο.

Με την συνάρητση insert εισάγουμε στον πίνακα κατακερματισμού το στοιχείο με τιμή n. Αν ο πίνακας έχει γεμίσει τότε διπλασιάζουμε το μέγεθος του πίνακα. Άρα η συνάρτηση κατακερματισμού αλλάζει. Άρα ξαναεισάγουμε τα στοιχεία στον πίνακα κατακερματισμού χρησιμοποιώντας την νέα συνάρτηση κατακερματισμού. Αυτό γίνεται με έναν βοηθητικό πίνακα data2.
Η εισαγωγή κάθε στοιχείου στον πίνακα κατακερματισμού γίνεται ως εξής: υπολογίζουμε την θέση th του στοιχείου στον πίνακα κατακερματισμού. Αν δεν έχει τοποθετηθεί αριθμός στην θέση position του πίνακα (δηλαδή αν ο πίνακας σε εκείνη την θέση έχει το -1) τότε τοποθετούμε σε εκείνη τη θέση το στοιχείο. Αν έχει τοποθετηθεί άλλο στοιχείο σε εκείνη τη θέση τότε γίνεται αναζήτηση για την επόμενη θέση στην οποία δεν έχει τοποθετηθεί κάποιο στοιχείο.
Αν κατά την αναζήτηση αυτή φτάσουμε στο τέλος του πίνακα τότε συνεχίζουμε από την αρχή του. Όταν βρούμε την θέση του στοιχείου τότε τοποθετούμε σε αυτή το στοιχείο.



-GETSIZE HASHTABLE

Στη main όταν σε κάποια γραμμή του αρχείου commands.txt εντοπιστεί η φράση GETSIZE HASHTABLE τότε εγγράφουμε στο αρχείο output.txt το μέγεθος του hashatble, το οποίο παίρνουμε με τη συνάρτηση getsize και στη συνέχεια αλλάζουμε γραμμή. Η συνάρτηση getsize υπολογίζει και επιστρέφει το πλήθος των στοιχείων του πίνακα κατακερματισμού. Συγκεκριμένα, η συνάρτηση έχει μία μετβαλητή pl_st που συμβολίζει το πλήθος των στοιχείων του πίνακα κατακερματισμού. Η pl_st αρχικοποιείται με 0. Για κάθε στοιχείο του πίνακα data αν το στοιχείο είναι διάφορο του -1 τότε το pl_st αυξάνεται κατά 1. Τέλος, επιστρέφεται το pl_st.



-SEARCH HASHTABLE number

Στη main, όταν σε κάποια γραμμή του αρχείου commands.txt εντοπιστεί η φράση SEARCH HASHTABLE τότε αποθηκεύουμε τον αριθμό που ακολουθεί στην ακέραια μεταβλητή number όπως έχουμε περιγράψει και στην INSERT MINHEAP. Εκτός από τον χρόνο εκτέλεσης εγγράφουμε στο output.txt και αν υπάρχει το number στο hashtable και στη συνέχεια αλλάζουμε γραμμή. Συγκεκριμένα, εγγράφουμε "SUCCESS" αν υπάρχει και "FAILURE" αν δεν υπάρχει. Αυτό το βρίσκουμε με τη συνάρτηση search της κλάσης hashtable.

Η συνάρτηση search έχει ως παράμετρο μία ακέραιη μεταβλητή n. Η search αναζητάει το στοιχείο με τιμή n στον πίνακα κατακερματισμού και αν το βρει επιστρέφει το string "SUCCESS". Αλλιώς επιστρέφει το string "FAILURE".

Σύμφωνα με την συνάρτηση κατακερματισμού που έχουμε επιλέξει, η θέση στην οποία θα έπρεπε να είναι το στοιχείο που αναζητάμε στον πίνακα κατακερματισμού είναι (3*n+7) mod maxsize.

> Αν η θέση στην οποία θα έπρεπε να είναι το n εχει το -1 (δηλαδή είναι κενή) τότε το στοιχείο που αναζητάμε δεν υπάρχει στον πίνακα κατακερματισμού άρα επιστρέφεται "FAILURE".
> Αν η θέση στην οποία θα έπρεπε να είναι το n δεν είναι κενή και δεν έχει γίνει σύγκρουση (δηλαδή περιέχει το στοιχείο που αναζητάμε) τότε επιστρέφει "SUCCESS".
> Αν η θέση στην οποία θα έπρεπε να είναι το n δεν είναι κενή και έχει γίνει σύγκρουση (δηλαδή δεν περιέχει το στοιχείο που αναζητάμε) τότε ακολουοθύμε τα εξής βήματα:
  -Αρχίζουμε από την θέση μετά από την θέση στην οποία θα έπρεπε να είναι το στοιχείο και ελέγχουμε με τη σειρά όλα τα επόμενα στοιχεία του πίνακα κατακερματισμού. Αν βρούμε το στοιχείο τότε επιστρέφουμε "SUCCESS". Αν βρούμε κενή θέση τότε επιστρέφουμε "FAILURE". Αν ο έλεγχος φτάσει στο τέλος του πίνακα τότε συνέχιζουμε τον έλεγχο από την αρχή του πίνακα. Αν έχουμε ελέγξει όλα τα στοιχεία και δεν έχουμε βρει το n τότε επιστρέφουμε "FAILURE".
